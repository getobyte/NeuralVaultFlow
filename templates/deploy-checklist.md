# Deploy Checklist Template

Template for NVC key `{project}:deploy-check-{date}` — pre-deployment readiness verification.

**Purpose:** Standardized deploy gate output that blocks deployment when CRITICAL items fail.

---

## File Template

```markdown
# Deploy Check: {project-name}

**Date:** {date}
**Environment:** {environment}
**Version:** {version}
**Result:** {result}

## Summary

| Category | Status | Critical Blockers |
|----------|--------|-------------------|
| Environment | {status} | {blocker-count} |
| Database | {status} | {blocker-count} |
| Build | {status} | {blocker-count} |
| Tests | {status} | {blocker-count} |
| Security | {status} | {blocker-count} |
| Health | {status} | {blocker-count} |
| Rollback | {status} | {blocker-count} |
| **Overall** | **{result}** | **{total-blockers}** |

---
```

## Field Documentation

| Field | Type | Required | Purpose | Example |
|-------|------|----------|---------|---------|
| `{project-name}` | variable | Yes | Project identifier | `my-saas-app` |
| `{date}` | variable | Yes | Check date | `2026-03-26` |
| `{environment}` | variable | Yes | Target environment | `production`, `staging` |
| `{version}` | variable | Yes | Version being deployed | `v2.3.1`, `abc1234` |
| `{result}` | variable | Yes | Overall result | `PASS`, `FAIL` |
| `{status}` | variable | Yes | Per-category status | `PASS`, `FAIL`, `WARN` |
| `{check}` | variable | Yes | Checkbox state | `x` (pass), ` ` (fail), `!` (warning) |
| `{blocker-count}` | variable | Yes | Critical failures per category | `0`, `1`, `2` |

## Section Specifications

### Summary Table
**Purpose:** Quick pass/fail overview per category.
**Quality check:** Overall result is FAIL if any category has critical blockers > 0.

### Category Sections
**Purpose:** Detailed checkbox items per concern.
**Quality check:** Every item has a clear pass/fail state. Notes explain any failures.

### Blockers Table
**Purpose:** Explicit list of what must be fixed before deploy.
**Quality check:** Only present when result is FAIL. Each blocker has a resolution action.
